# Pr1.MinWebService — Мини веб-служба

Каталог товаров с хранением в памяти процесса. Практическое занятие №1 по дисциплине «Технологии разработки приложений на базе фреймворков».

## Запуск
```bash
dotnet run
```

Служба запустится на `http://localhost:54255`
## Примеры запросов (curl)

### Создать элемент
```bash
curl -X POST http://localhost:54255/api/items \
  -H "Content-Type: application/json" \
  -d '{"name": "Мышь Logitech", "price": 1999.90}'
```

Ответ `201 Created`:
```json
{"id":"...","name":"Мышь Logitech","price":1999.90}
```

### Получить список элементов
```bash
curl http://localhost:54255/api/items
```

### Получить элемент по ID
```bash
curl http://localhost:54255/api/items/{id}
```

### Ошибка валидации (пустое имя)
```bash
curl -X POST http://localhost:54255/api/items \
  -H "Content-Type: application/json" \
  -d '{"name": "", "price": 100}'
```

Ответ `400 Bad Request`:
```json
{"code":"validation","message":"Поле name не должно быть пустым","requestId":"..."}
```

### Ошибка валидации (отрицательная цена)
```bash
curl -X POST http://localhost:54255/api/items \
  -H "Content-Type: application/json" \
  -d '{"name": "Тест", "price": -5}'
```

### Элемент не найден
```bash
curl http://localhost:54255/api/items/11111111-1111-1111-1111-111111111111
```

Ответ `404 Not Found`:
```json
{"code":"not_found","message":"Элемент не найден","requestId":"..."}
```

## Архитектура

### Конвейер обработки запросов (Middleware Pipeline)

Запрос проходит через три обработчика в фиксированном порядке:
```
Клиент → RequestIdMiddleware → ErrorHandlingMiddleware → TimingAndLogMiddleware → Endpoint
```

1. **RequestIdMiddleware** — назначает каждому запросу уникальный идентификатор (GUID). Если клиент передал заголовок `X-Request-Id`, используется он; иначе генерируется новый. Идентификатор сохраняется в `HttpContext.Items` и возвращается клиенту в заголовке ответа.

2. **ErrorHandlingMiddleware** — оборачивает всю дальнейшую обработку в `try/catch`. Доменные исключения (`NotFoundException`, `ValidationException`) превращаются в ответы с соответствующим HTTP-кодом и единым JSON-форматом `{code, message, requestId}`. Непредвиденные исключения возвращают `500` без раскрытия деталей.

3. **TimingAndLogMiddleware** — запускает секундомер перед передачей запроса дальше, а после — записывает в журнал метод, путь, статус и время выполнения в миллисекундах.

### Слои приложения

| Слой | Папка | Назначение |
|------|-------|------------|
| Домен | `Domain/` | Неизменяемые записи: `Item`, `CreateItemRequest`, `ErrorResponse` |
| Ошибки | `Errors/` | Иерархия исключений: `DomainException` → `NotFoundException`, `ValidationException` |
| Сервисы | `Services/` | `IItemRepository` / `InMemoryItemRepository` — хранение в `ConcurrentDictionary`; `RequestId` — генерация идентификатора запроса |
| Middleware | `Middlewares/` | Три обработчика конвейера |
| Модели | `Models/` | DTO для привязки входных данных |

### Почему это каркас, а не случайный набор функций

Каждый компонент решает одну задачу и не зависит от деталей реализации других. Middleware подключаются в `Program.cs` в явном порядке, и любой из них можно заменить или убрать без влияния на остальные. Хранилище скрыто за интерфейсом `IItemRepository`, поэтому переход с in-memory на базу данных потребует только новой реализации без изменения эндпоинтов. Валидация и обработка ошибок разделены: валидация бросает типизированное исключение, а middleware его ловит и формирует ответ — это избавляет эндпоинты от дублирования логики обработки ошибок.

### Правила предметной области

- Имя товара не может быть пустым или состоять только из пробелов
- Имя товара не длиннее 100 символов
- Цена не может быть отрицательной

### Безопасность от конфликтов

`ConcurrentDictionary` обеспечивает потокобезопасное создание и чтение элементов при параллельных запросах.
